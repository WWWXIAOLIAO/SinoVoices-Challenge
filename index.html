赛事简介

近年来，语音识别技术在标准普通话场景中取得了显著进展，并已广泛应用于智能语音助手、会议转写、智能客服和语音交互等实际场景。然而，真实中文语音输入往往并不局限于标准普通话，而是广泛包含粤语、闽南语、客家话、潮汕话等多种区域性方言，以及带有地域特征的口音表达。

中文方言具有地域分布广、语音差异大、词汇表达多样、标注成本高和资源建设不均衡等特点。相比普通话语音数据，方言语音数据普遍存在资源稀缺、覆盖不足和标注规模有限等问题，是典型的低资源语音识别场景。

ChinaVoices Challenge 2026 拟以 NCMMSC 2026 为平台，围绕中文多方言语音识别与中文多方言种类识别任务，构建统一的评测数据、评价指标和竞赛流程，为学术界和工业界提供公开、公平、可复现的中文多方言语音处理评测平台。

覆盖多方言

计划发布覆盖十八种中文方言及区域性口音的人工标注语音数据，支撑低资源方言建模与跨地域泛化评估。

双任务设置

设置中文多方言种类识别与中文多方言语音识别两个核心任务，分别评估方言类别判别能力与语音内容转写能力。

双排行榜机制

设置公开数据排行榜和开放数据排行榜，兼顾正式排名公平性与技术探索价值。

任务与赛道

任务一：中文多方言种类识别

任务目标：参赛者需要根据输入语音判断其所属的方言类别或地域口音类别。

任务重点：关注系统对不同方言种类的区分能力，尤其关注相近方言混淆、地域口音差异和低资源方言种类识别等问题。

初步评价指标：准确率 Accuracy。

任务二：中文多方言语音识别

任务目标：构建面向中文多方言语音的自动语音识别系统，将输入语音准确转写为对应文本。

任务重点：评估模型在不同方言类别、地域口音、说话人和声学条件下的转写能力，重点关注多方言场景中的识别准确率。

初步评价指标：字符错误率 CER。

数据集说明

本挑战赛计划基于组织方整理和构建的中文多方言语音数据开展统一评测。数据将覆盖多种中文方言和区域性方言语音，后续将根据数据质量、类别规模、标注一致性、语言学划分和竞赛可操作性，对方言类别进行整理、合并和规范化。

  • 覆盖多种中文方言和区域性方言语音；
  • 包含不同地区、不同说话人和不同发音习惯；
  • 支持中文多方言种类识别和中文多方言语音识别两个任务；
  • 计划为每个语种/方言提供约 3 小时参考数据（reference data，最终规模以发布版本为准）；
  • 公开测试集 / 公开评测集（open test set / open evaluation set）将按重要日期发布，用于阶段性提交与系统调试；
  • 最终排名基于隐藏测试集 / 隐藏评测集（hidden test set / hidden evaluation set），该集合不公开发布；
  • 提供统一的数据划分方式、标注规范、提交格式和评测脚本。

常见问题

Q：可以只参加一个任务吗？

A：可以。参赛队伍可根据自身研究方向选择参加中文多方言种类识别任务、中文多方言语音识别任务，或同时参加两个任务。

Q：公开数据排行榜和开放数据排行榜有什么区别？

A：公开数据排行榜用于正式排名和奖项评定，数据与模型使用范围受限；开放数据排行榜主要用于探索性能上限，允许使用更多合法数据资源，但需要披露来源和使用方式。

Q：是否可以使用公开预训练模型？

A：可以。公开数据排行榜中允许使用公开可获得、可复现的预训练模型；所有外部资源、数据增强方法和后处理策略均需在系统说明中明确列出。

Q：参考数据、公开测试集和隐藏测试集有什么区别？

A：参考数据（reference data）将按语种/方言类别发布，计划每类约 3 小时，用于参赛队伍训练和开发；公开测试集 / 公开评测集（open test set / open evaluation set）将按时间线发布，用于阶段性提交与系统调试；隐藏测试集 / 隐藏评测集（hidden test set / hidden evaluation set）不公开发布，用于最终排行榜和系统排名。

数据使用规则

公开数据清单与数据登记机制

组织方将建立“中文方言语音开源数据公共清单”，列出参赛者可以使用的公开数据资源，包括数据名称、语种/方言类别、数据规模、许可协议、下载地址和适用任务等信息。

若参赛队伍使用未在公共清单中的公开数据，可向组织方提交数据登记申请。经确认该数据为公开可获得、合法可使用且符合竞赛规则后，该数据将被补充到公共清单中，供其他参赛队伍参考。

提交材料与合规检查

所有参赛队伍在提交系统结果时，均需提交数据使用声明，说明所使用的训练数据、外部数据、预训练模型、数据增强方法和后处理策略。对于公开数据排行榜，组织方将对数据使用情况进行合规性检查；对于开放数据排行榜，参赛队伍需尽可能披露数据来源、规模和类别覆盖情况，以便后续进行公平分析和结果解释。

竞赛规则

所有参赛队伍需遵守挑战赛规则。组织方保留对参赛条款、数据使用规则和最终排名解释、修改与补充的权利。

1. 公开数据排行榜

  • 仅允许使用组织方提供的数据、明确允许使用的公开开源数据、公开预训练模型以及公开可复现的训练资源。
  • 参赛队伍提交结果时，需要同时提交数据使用说明，列明训练数据、预训练模型、数据增强方法和外部资源。
  • 该排行榜用于确定挑战赛最终正式排名、奖项评定和证书。

2. 开放数据排行榜

  • 允许使用公开数据、企业内部数据、实验室自建数据或其他合法可使用的数据资源。
  • 参赛队伍需在系统说明中明确列出所使用数据的来源、规模、类别覆盖和使用方式。
  • 该排行榜主要用于展示统一公开评测条件下的性能上限，不直接等同于奖金或正式排名。

3. 论文与复现

  • 挑战赛系统描述论文将综合考虑论文质量和排行榜成绩。
  • 最终奖项评定原则上以公开数据排行榜成绩为主要依据，同时综合考虑系统描述论文质量、方法创新性、实验完整性和结果可复现性。
  • 组织方可根据需要对高排名队伍的提交材料进行合规性检查与结果核实。
